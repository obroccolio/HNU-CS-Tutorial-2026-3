# 题目 d) - 矩阵转置对性能的影响 完整分析报告

## 一、实验概述

本实验对 **6 种循环顺序** 分别在 **-O1、-O2、-O3** 三个编译优化级别下，测试**转置前后**的性能影响。

**实验参数**：
- 矩阵大小：1001 × 1001
- 编译器：GCC 14.2.0 with `-march=native`
- 平台：Apple Silicon M-series MacBook
- 总实验数：6 循环顺序 × 3 优化级别 = 18 个版本

## 四、核心发现

### 1️⃣ **转置的双面性**

#### 转置有益（应该转置）：
- **ijk / jik** ：在所有优化级别都获得加速
  - O1：+4~7%（小幅加速）
  - O2：+41%（显著加速）⭐ **最大收益**
  - O3：+40%（显著加速）

- **jki** ：中等收益
  - O1：+12.72% ✓
  - O2：+16.06% ✓
  - O3：+10.89% ✓

#### 转置有害（不应该转置）：
- **kij** ：最严重的"负优化"
  - O1：-15.76%（性能下降）
  - O2：-51.59%（性能严重下降）⭐ **最差表现**
  - O3：-50.89%（性能严重下降）

- **ikj** ：中等有害
  - O1：-7.80%
  - O2：-40.56%
  - O3：-45.48%

- **kji** ：轻微有害
  - 各级别稳定在 -7~11%

### 2️⃣ **编译优化的非线性影响**

```
性能对比（以O1为基准）
-O1 → -O2 的跳跃：
  ijk: 782.85 → 783.06（基本无变）→ 转置后: 728.89 → 457.40（大幅改善！）
  kij: 299.21 → 187.27（大幅加速）→ 转置后变慢的反差更大

结论：编译优化使得"正优化"和"负优化"的效果都被放大
```

### 3️⃣ **转置开销 vs 计算收益**

转置开销极小（0.8~1.7ms），可以忽略不计：
- **jki (O1)**：转置开销0.94ms，计算节省84.55ms，**投资回报率 89.8x** ✓✓✓
- **kij (O2)**：转置开销1.00ms，计算额外损耗96.62ms，**负投资回报率** ❌❌❌

## 五、题目 d) 的答案

### 问题：矩阵转置对性能的影响

**答案**：

转置对性能的影响**高度依赖于循环顺序和编译优化级别**：

1. **某些循环顺序（ijk/jik）中，转置带来显著加速**
   - 尤其在 -O2/-O3 下，性能提升 40%+
   - 原因：原始循环中 B 是列优先访问（步长=N），转置后变成行优先（步长=1）
   - 编译器能更好地向量化行优先访问

2. **某些循环顺序（kij/ikj）中，转置反而减速**
   - 减速幅度 -15% ~ -51%
   - 原因：原始循环中 B 已是行优先访问（最优），转置破坏了这一特性
   - 这被称为"负优化"现象

3. **编译优化级别放大了这一效果**
   - -O1 下的差异较小（几个百分点）
   - -O2/-O3 下的差异巨大（40%+）

### 最佳实践

| 循环顺序 | 推荐 | 性能提升 |
|---------|------|---------|
| ijk | ✓ 转置 | +40% (O2/O3) |
| jik | ✓ 转置 | +40% (O2/O3) |
| jki | ✓ 转置 | +12% (O1-O3) |
| ❌ kij | ✗ 不转置 | 保持最优 |
| ❌ ikj | ✗ 不转置 | 保持最优 |
| ❌ kji | ✗ 不转置 | 保持最优 |
