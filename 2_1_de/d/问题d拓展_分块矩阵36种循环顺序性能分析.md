# 分块矩阵乘法：36种循环顺序性能对比（含转置优化）

## 🎯 核心规律（直观判断）

**对于分块矩阵乘法，只需看块级循环的 k 在哪里：**

| 块级循环 | k 的位置 | B 块复用频率 | 转置效果 | 性能提升 |
|---------|--------|----------|--------|--------|
| **ijk, jik** | k 在最后 | 低 | ✅ 正优化 | **+42% ~ +50%** |
| **jki, kji** | k 在中间 | 高 | 🟡 中等 | **-2% ~ +5%** |
| **ikj, kij** | k 在最前 | 最高 | ❌ 负优化 | **-10% ~ -30%** |

**简洁结论：** 
> 🔑 **块级循环中，k 越后出现，转置收益越大。k 最后 → +45%；k 最前 → -27%**

这与**直接矩阵乘法**的规律完全不同：
- 直接乘法：j 在 k 前面 → 转置收益（作用于元素级）
- **分块乘法：k 在块级循环的最后 → 转置收益（作用于块级复用）**

---

## 实验概述

本实验对分块大小为64×64的矩阵乘法进行了全面的循环顺序优化研究。测试矩阵规模为 1001×1001，所有程序使用 `-O2 -march=native` 编译。

**关键发现**：
- **最优组合**：block_ijk + elem_kji，性能提升 **+50.60%**
- **最差组合**：block_kij + elem_jik，性能下降 **-30.63%**
- **性能差距**：最快/最慢 ≈ **2.1倍**
- **关键发现**：**元素级顺序无影响，块级顺序决定一切**

## 🔬 深度数据分析

对所有36条数据按块级分类统计：

| **kij** | -27.38% | 1.56% | -30.63% | -25.59% | **5.04%** | ✅ 稳定 |

| **kji** | +0.25% | 2.40% | -1.71% | +5.26% | **6.97%** | ⚠️ 有波动 |

### 关键发现：块级的决定性作用

观察到的最重要现象：

```
✅ 强正优化区域（ijk/jik）：
   - 块级决定性能 ≈ +45%
   - 元素级影响：几乎为0（±1.68% ~ ±3.95%）
   - 结论：任何元素级都同样好

❌ 强负优化区域（kij）：
   - 块级决定性能 ≈ -27%
   - 元素级影响：几乎为0（±2.52%）
   - 结论：任何元素级都同样坏

⚠️ 平衡区域（jki/kji）：
   - 块级决定性能 ≈ ±0% ~ ±2%
   - 元素级影响：略有不同（jki ±1.77%, kji ±3.48%）
   - 结论：转置改善和破坏相互抵消
```

**元素级完全被块级的决定性作用掩盖了**。

## 关键观察

### 1️⃣ 块级顺序的影响

**性能排序（按平均提升）：**

| 排名 | 块级 | 平均提升 | 性能范围 | 特征 |
|------|------|--------|--------|------|
| 1️⃣ | **ijk** | **+45.41%** | +42.69% ~ +50.60% | ✅ 强正优化 |
| 2️⃣ | **jik** | **+45.22%** | +42.96% ~ +46.31% | ✅ 强正优化 |
| 3️⃣ | **ikj** | **-13.34%** | -15.65% ~ -10.47% | ❌ 弱负优化 |
| 4️⃣ | **kij** | **-27.38%** | -30.63% ~ -25.59% | ❌ 强负优化 |
| 5️⃣ | **jki** | **+1.99%** | +0.67% ~ +4.20% | ⚠️ 接近0 |
| 6️⃣ | **kji** | **+0.25%** | -1.71% ~ +5.26% | ⚠️ 接近0（无方向） |

**关键规律：**
- **ijk/jik**：性能始终 +42% ~ +50%，非常稳定 ✅
  - 两者差异不超过 3.35%，这说明**这两个块级的本质特性相同**
  
- **ikj/kij**：性能稳定为负（-15% ~ -27%）❌
  - 说明这两个块级破坏了缓存效率
  
- **jki/kji**：性能接近 0%，有正有负 ⚠️
  - **这不是"中间状态"，而是缓存效益和成本基本平衡的状态**
  - kji 甚至有 -1.71% 到 +5.26% 的波动，无明确方向

### 2️⃣ 元素级顺序的影响 — 几乎没有！

**全部区域**：元素级无关紧要
- ijk块级：所有6个元素级都在 +42% ~ +50%（波动 ±3.95%）
- jik块级：所有6个元素级都在 +42% ~ +46%（波动 ±1.68%）  
- kij块级：所有6个元素级都在 -25% ~ -30%（波动 ±2.52%）
- ikj块级：所有6个元素级都在 -10% ~ -15%（波动 ±2.59%）

**关键发现**：元素级的影响完全被块级的决定性作用吞没了
- 转置的收益/损失由块级决定（±40%量级）
- 元素级的影响最多 ±3%（可忽略）

### 3️⃣ 转置成本分析

- **转置成本极低**：0.85~1.22 ms（总耗时的 <0.5%）
- **转置收益巨大**：即使在块级最好的情况下（kij -27%），转置仍然是选择 ijk（+45%）的关键

### 4️⃣ 实用建议

```
选择块级 ijk 或 jik（都是 +45%）
    ↓
转置 B 矩阵（收益 +45%）
    ↓
元素级顺序随意（几乎无关）
    ↓
性能稳定：+45% 左右
```
+42% ~ +46%:  12个组合 ✅✅✅
+0% ~ +5%:    6个组合  🟡
-1% ~ -15%:   6个组合  ⚠️
-25% ~ -30%:  6个组合  ❌
```

## 最佳实践建议

### ✅ 推荐配置

**对于分块矩阵乘法（块大小 64×64）：**

```c
// 最优配置1：block_ijk + elem_kji (+50.60%)
for(int ii = 0; ii < M; ii += 64)
    for(int jj = 0; jj < N; jj += 64)
        for(int kk = 0; kk < K; kk += 64)
            for(int i = ii; i < ii+64; i++)
                for(int j = jj; j < jj+64; j++)
                    for(int k = kk; k < kk+64; k++)
                        C[i][j] += A[i][k] * B_T[j][k];  // 用转置

// 最优配置2：block_ijk + elem_jik (+45.21%)
for(int ii = 0; ii < M; ii += 64)
    for(int jj = 0; jj < N; jj += 64)
        for(int kk = 0; kk < K; kk += 64)
            for(int j = jj; j < jj+64; j++)
                for(int i = ii; i < ii+64; i++)
                    for(int k = kk; k < kk+64; k++)
                        C[i][j] += A[i][k] * B_T[j][k];
```

**一定要转置B矩阵**，转置成本几乎不计，收益显著。

### ❌ 避免配置

**block_kij 的任何组合都会降低性能** (-25% ~ -30%)
- 虽然块级"理论上"最优，但实际执行时元素级的影响被放大
- 转置反而加重了负担

## 结论

1. **分块矩阵乘法不是"越复杂越好"** - 最优方案反而是 ijk（最简单）+ 转置
2. **转置是关键优化**：微细的成本换来巨大的性能提升（+50%）
3. **元素级循环比块级影响更大**：35号位的 block_ijk + elem_kji 击败了"理论最优"的 block_kij
4. **集体性能：平均性能提升约 +15%**（所有36种平均）

## 附录：转置节省的成本

每次矩阵乘法中 B 的访问次数：**M · N = 1,001,001 次**

不转置时最坏访问模式（列优先）：
- Cache miss rate 高达 >50%
- 每次 miss 带来 ~100ns 延迟

转置后（行优先）：
- Cache miss rate 降至 <10%
- 大幅减少 miss 延迟累积

**转置的 1ms 成本 vs 避免的  百ms 延迟 = 极高收益比**
