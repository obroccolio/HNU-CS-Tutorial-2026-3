## 幻灯片 1：引入与主题聚焦

今天我们小班讨论课的选题是 2.1，主题是“循环顺序之争” (Matrix Multiplication Loop Order) 。我们将重点探讨矩阵乘法中六种不同循环顺序的 Cache 友好性，并定位性能瓶颈 。具体来说，我们会从 (b) 访问模式和 (c) Cache Miss 两个维度展开深度剖析 。

---

## 幻灯片 2：实验现象与直觉类比

* 在开始理论分析前，我们先看一组实验现象 。对于同样的 C 语言代码、完全相同的矩阵乘法算法，仅仅是交换了循环的嵌套顺序，性能就出现了天壤之别 。
- 最快的顺序是 `ikj`，耗时仅为 0.204 秒，算力达到 9.84 GFLOPS 。

- 而最慢的顺序是 `jki`，耗时高达 0.741 秒，算力仅有 2.71 GFLOPS 。

- 两者之间的性能差距达到了惊人的 3.6 倍 。

为什么同一个算法会有这么大的速度差 ？我们可以用生活中的场景来类比 ：去图书馆找 10 本书，如果它们都在同一个书架上，你只需要顺序访问，很快就能拿完；但如果这 10 本书散落在 10 层楼的不同角落，属于随机访问，你的效率将极其低下 。这就是空间局部性的魔力，也是我们要探究的核心原因 。

---

## 幻灯片 3：基础知识（内存存储与 Cache Line）

* 为了解释这种差异，我们需要回顾两个基础知识点 。
- **行优先存储：** C 语言采用的是行优先存储 (Row-Major Order) 机制，即矩阵的每一行在物理内存中是连续存放的 。这意味着 `A[i][j]` 的物理地址后面紧跟着的就是 `A[i][j+1]` 。如果你要跨行访问，比如先访问 `A[i][j]` 再访问 `A[i+1][j]`，CPU 就需要在内存中跨越 N 个元素的距离 。

- **Cache Line (缓存行)：** CPU 每次从内存取数据时，并不是只拿一个元素，而是搬运一整块，通常是 64 字节 。如果你顺着内存拿（具有良好的空间局部性），搬一次可以连续用好几次；如果是跳着拿，搬进来一整块却只用其中一个元素，效率就极低 。

---

## 幻灯片 4 & 5：深度剖析（谁在内层决定访问模式）

**【主讲人引导语】** 接下来我们进行深度剖析 。这 6 种写法（`ijk`、`ikj`、`jik`、`jki`、`kij`、`kji`）的本质，是把 `i`、`j`、`k` 三个变量分别放在了外层、中层和最内层 。而核心关键在于：谁在最内层，直接决定了程序是“顺着内存走”还是“跳着走” 。 在核心计算代码 `C[i][j] += A[i][k] * B[k][j];` 中，内层变量变化最快 。我们以 `ijk` 顺序为例，此时 `k` 在最内层 ：

- 对于 `A[i][k]`，它是顺着 `k` 扫描行的，属于行优先（Stride-1），对 Cache 友好 。

- 对于 `B[k][j]`，它是顺着 `k` 扫描列的，属于列优先（跨度为 N），对 Cache 不友好 。

- 对于 `C[i][j]`，由于它与 `k` 无关，在最内层循环中是一个固定点，可以复用 。

换句话说，同样的计算，内层循环的不同导致了时间上可能存在一个数量级的差距 。

## 幻灯片 8&9：六种顺序的模式总表与“Cache 真爱”

* 基于上述分析，我们总结出了六种顺序的访问模式表 ：
- 当 `k` 在内层（`ijk` / `jik`）时：A 是行优先，B 是列优先，C 是固定点，Cache 友好度一般 。

- 当 `i` 在内层（`jki` / `kji`）时：A 和 C 都是列优先，B 是固定点，Cache 友好度极差 。

- 当 `j` 在内层（`ikj` / `kij`）时：A 是固定点，B 和 C 都是行优先，Cache 友好度极佳 。

这就是为什么 `ikj` 是最快的 。因为它的最内层是 `j`，相当于顺着内存地址扫一整行 。`B[k][j]` 和 `C[i][j]` 都是连续访问，而 `A[i][k]` 则是固定在寄存器中 。这种模式不仅完美利用了 Cache Line，还让硬件预取器能够完美预测下一个数据并提前装载，CPU 几乎不需要等待内存搬运 。

## 幻灯片 10-13：量化分析（Cache Miss 与性能瓶颈）

 我们已经从直觉上理解了访问模式，最后一部分 (c)，我们将用硬件计数器来量化这种直觉 。为什么耗时会差这么多？因为 Cache miss 导致需要等待内存读取，进而引发流水线空转，最终导致 CPI (Cycles Per Instruction) 显著上升 。 我们对比最快的 `ikj` 和最慢的 `jki` ：

- **L1 Cache:** `ikj` 每 8 次迭代发生 2 次 Miss，Miss Rate 为 12.5% ；而 `jki` 每次访问都是跨列，空间局部性为 0，Miss Rate 高达 100% 。两者差值约为 87.5% 。

- **L2 Cache:** `ikj` 凭借步长为 1 的模式让硬件预取器大显身手，Miss Rate 小于 1% ；但预取器很难处理 `jki` 这种大跨度的模式，导致其 L2 Miss Rate 高达 90%−95% 。两者差值在 90% 以上 。

- **L3 Cache:** `ikj` 几乎完全命中 (0.01%) ；`jki` 由于效率极低、换入换出频繁，Miss Rate 在 5%−20% 左右 。

**最终结论：** 真正的性能瓶颈在于 L1/L2 缓存的带宽与冲突不命中 。列访问（Stride-N）不仅让缓存利用率极低，还让硬件预取机制完全失效，导致 CPU 被迫经历长达几百个周期的停顿 。
