# 选题 2.1 循环顺序之争： (b)(c) 分析（结合 gemm_loop_order.c）

## 代码锚点

- 目标公式：`C[i][j] += A[i][k] * B[k][j]`，见 [gemm_loop_order.c:L34-L41](file:///d:/系统讨论课/gemm_loop_order.c#L34-L41)
- 6 种循环顺序实现：见 [gemm_loop_order.c:L36-L82](file:///d:/系统讨论课/gemm_loop_order.c#L36-L82)
- 行优先存储前提：`double A[N][N]` 在 C 中为 row-major（同一行内 `A[i][0..N-1]` 连续）

## (b) A/B/C 的访问模式与 Cache 友好性

### 判定规则（上课用语对齐）

- 行优先（Row）：内层循环变量沿着“列下标”变化（如 `A[i][k]` 中 `k` 在内层），访存连续，空间局部性好
- 列优先（Col）：内层循环变量沿着“行下标”变化（如 `A[i][k]` 中 `i` 在内层），跨行步长约为 `N*sizeof(double)`，空间局部性差
- 固定（Fixed）：该数组元素在内层循环中下标不变（常见可进寄存器/被反复读写同一 cache line）

### 逐顺序访问模式表（回答 b 的核心）

| 循环顺序  | 内层循环 | A 访问                   | B 访问              | C 访问              | 直觉上的 Cache 友好性         |
| ----- | ---- | ---------------------- | ----------------- | ----------------- | ---------------------- |
| `ijk` | `k`  | Row (`A[i][k]`)        | Col (`B[k][j]`)   | Fixed (`C[i][j]`) | B 列访问拖后腿，通常不够好         |
| `ikj` | `j`  | Fixed (`A[i][k]`)      | Row (`B[k][j]`)   | Row (`C[i][j]`)   | 通常最友好：B/C 连续，A 可复用     |
| `jik` | `k`  | Row                    | Col               | Fixed             | 与 `ijk` 类似，B 仍是列访问     |
| `jki` | `i`  | Col (`A[i][k]`)        | Fixed (`B[k][j]`) | Col (`C[i][j]`)   | 常见最差：A/C 都跨行步进         |
| `kij` | `j`  | Col（但每次只取一个 `A[i][k]`） | Row               | Row               | B/C 连续；A 为跨行点取，介于好与一般  |
| `kji` | `i`  | Col                    | Fixed             | Col               | 与 `jki` 同类：A/C 跨行步进，偏差 |

### 哪种顺序对 Cache 最友好（给出可直接上 PPT 的结论）

- 通常最友好：`ikj`
  - 内层 `j` 扫一整行：`B[k][j]` 连续、`C[i][j]` 连续
  - `A[i][k]` 在内层固定，可用寄存器保存，减少访存压力
- 通常最不友好：`jki` 与 `kji`
  - 内层为 `i`，导致 `A[i][k]` 与 `C[i][j]` 在内层跨行步进（stride 约 `N*8` 字节）
  - 每次只用到一个 `double`，却可能把整条 cache line（典型 64B）带进来，空间局部性利用率低

## (c) 最快与最慢的 L1/L2/L3 miss rate 差异与瓶颈定位（方法 + 预期）

### 为什么不同循环顺序会影响 miss rate

- 关键差异来自“内层循环是否连续走内存”
  - 连续走（Row）：同一条 cache line 被充分利用（空间局部性）
  - 跨行步进（Col）：cache line 只用到 1 个元素就被抛弃（空间局部性极差），更容易把低层 cache 也冲爆
- 对 `N≈1000` 的三矩阵，A/B/C 各约 `1001^2*8 ≈ 8 MB`，三者合计约 24MB
  - 不可能放进 L1/L2，必须依赖 L3 + 良好的访问模式来“让正在用的一小段数据留在上层”

### 预期结论

- 最快（常见 `ikj`）：
  - L1 miss rate 显著更低（B/C 行连续，且存在短期复用）
  - L2/L3 miss rate 也更低（因为上层命中率高，向下级请求少）
- 最慢（常见 `jki` 或 `kji`）：
  - L1 miss rate 很高（A/C 跨行步进）
  - L2 miss rate 也会偏高（访问看起来更“随机”，更难被硬件预取器有效捕捉）
  - L3/LLC miss 是否极高取决于机器：若 LLC 足够大，很多访问会落在 L3 hit；若 LLC 较小/冲突严重，会进一步掉到内存

### 哪一层 Cache 是瓶颈（怎么从数据读出来）

- 先看“性能差距主要来自哪里”：对比最快 vs 最慢的每层 miss rate 与总耗时
  - 若 L1 miss rate 差异巨大且最慢的 CPI 明显上升，通常瓶颈在 L1D（加载/存储受阻，流水线空转）
  - 若 L1 miss 都不低但 L2 miss 差异才拉开，瓶颈多在 L2（L1 频繁去 L2 取数）
  - 若 L2 miss 与 LLC miss 差异最显著，且带宽接近峰值，瓶颈可能在 LLC/内存带宽
- 经验上：未分块的朴素 GEMM 在大矩阵下常“受内存层次结构支配”，`jki/kji` 往往把问题放大到 L2/LLC
